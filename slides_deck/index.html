Project Jupyter

Project Jupyter developed out of the IPython Project.

 

Project Jupyter develops tools that are language-agnostic.

 

Project Jupyter is non-profit, open-source, and free.

 

Jupyter originally stood for Julia, Python, and R, but now works with many other languages.

  • very popular with academic researchers
  • over 3 million public Jupyter Notebooks on GitHub

Limitations of Jupyter Notebooks

  • Notebooks don't do well with version control conflicts
  • Notebooks keep large variables in memory
  • Code can only be run chunk by chunk, not line by line
  • Hard to keep track of large code blocks
  • Unlike scripts, notebooks cannot be combined into computational pipelines.

Limitations of Jupyter Notebooks

  • Users (especially beginning coders) can lean on notebooks and never learn proper script-writing and code-testing techniques.
  • Encourages notebook development rather than software development
    Counterpoint: makes these tools available for more researchers who aren't going to develop the skills of a software developer or who don't have the funds to hire a software developer

Jupyter Notebook will soon cease to be supported as a stand-alone program, forcing all notebook users to switch to JupyterLab (which includes notebooks as one feature)

 

Notebooks are probably still the main draw of JupyterLab, but there are many new features outside of the notebooks.

 

JupyterLab

 
  • browser based (local host), extendable IDE
  • over 100 kernels are available for different programming languages
 

Contains 4 main tools:

  • consoles for interactive coding in the kernels of your choice
  • a text editor with syntax highlighting, key maps, indentation preferences, etc.
  • notebooks in your kernels of choice
  • a terminal (bash or PowerShell)
 

Additional features

  • File tree
  • Organize your Lab in either tabs or tiles
  • Export notebooks to more file types (LaTex, executable script, pdf, html, slides)
  • Opens most file types inside the browser tab (jpg, png, pdf, gif, json)
  • Opens large csv files (even larger than Excel can handle)
  • Has new extensions coming out regularly for features you will see in other IDEs, like GitHub, Vim, and LaTex, but also features that you won't find in other IDEs

Example of interactive widget

 
 
 
 

There currently isn't a straightforward way to export your interactive widgets to retain their interactive nature outside of a notebook. It can be done, but it's not seamless yet.

  • nbinteract converts notebooks to interactive html pages with embedded JavaScript libraries. Pages can then be hosted on the web (ex. GitHub Pages). But, some features require torubleshooting or may not work.
  • Voila is a package to convert and host notebooks containing widgets. 
  • Anvil lets you turn a notebook into an app, design a user interface, and host the app.

Limitations of JupyterLab

  • Some tools have been developed specifically for Python, and not other languages
  • Some kernels and extensions have multiple dependencies and can be complicated to install
  • Doesn't have the layout options of Shiny
  • Doesn't have the hosting abilities of Shiny and is more difficult to convert to interactive html
  • Some users might not want to have all their programming needs met by one tool
 
  • a multi-user hub that spawns multiple instances of the single user Jupyter Notebook server
  • requires SSL encryption and allows for Authentication
 

JupyterHub

Uses of JupyterHub for Research

  • Test out HPC code
  • Access predesigned environments with kernels that you can't/don't want to set up on your own machine
  • Conduct exploratory data analysis and data visualization on huge data sets
  • Use programs that leverage parallel computing for machine learning and deep learning, like Spark and TensorFlow

Uses of JupyterHub for Research

  • Teach courses by giving an entire class access to the same notebook environment
  • Use nbgrader to automatically grade assignments that are submitted over the hub

Support for JupyterHub for Research

  • Jupyter at Research Facilities forum
  • Jupyter in Education Mailing List
  • List of some institutions using JupyterHub
  • JupyterHub roadmap
 

When I would recommend JupyterLab

Beginning coders

  • browser/tab based, so familiar to new coders
  • free, included in Anaconda, works well with both Macs and PCs
  • clean, uncomplicated interface
  • has terminal, text editor, console
  • browser based gives easy access to Google
  • It was used in some of the beginning and next steps Python workshops this summer through RCS, so some beginners at NU will already be familiar

Intermediate coders

  • multiple languages can be read, written, and run from one IDE
  • over 3 million notebooks are available online to learn new skills and borrow code

Researchers who already use Python

(or who don't use R)

  • Combine text, code, and visuals in one narrative with Jupyter Notebooks
  • Create interactive software tools for analysis and visualization that can be made available to other researchers

All researchers

  • Access Google Drive files (or a remote server) shared by lab members or coauthors
  • Use data visualization tools and machine learning on huge data sets with HPC through JupyterHub

Teaching

  • Create interactive notebooks to supplement coursework with hands-on data analysis
  • Create interactive tutorials for programming skills for beginning or advanced topics
  • Publish personalized textbooks

 

Teaching (with advanced computational skills)

  • Utilize JupyterHub
 

Summary

 

JupyterLab brings the well-loved notebooks into a clean, simple IDE that can be expanded with extensions.

 

Perks are the included terminal, ability to code in multiple language consoles, an extension to access files on Google Drive, and the potential for JupyterHub.

 

Negatives include the notebooks don't encourage good coding habits and the interactive notebooks don't yet rival Shiny apps.

 

Researchers who already love notebooks, and who will soon be forced to used JupyterLab, will have an easy transition, as the design is minimalist.

 

More advanced Python coders who conduct research in all disciplines can feel confident that the functionality of JupyterLab will continue to improve as new features are added by Jupyter and new extensions are developed by users.
